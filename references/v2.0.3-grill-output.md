# jsn v2.0.3 — Grill Output

**Date:** 2026-06-09
**Participants:** Jace, Holly

## Goal

Release jsn v2.0.3 fixing the auth status false-positive issue and profile management discoverability.

## Issues

- #86 — `jsn auth status` returns `authenticated: true` even when the PDI has been released
- #87 — Profile management is unclear — no discoverable way to add or create a profile
- #88 — `jsn --profile` (no arg) shows help; no `jsn profiles create` subcommand

## Decisions

### Auth Status Fix (#86)

| Decision | Resolution |
|----------|------------|
| Mechanism | Two-layer check: time heuristic + live API verification |
| Layer 1 | Stamp `last_seen` on every successful API call (login + dev/data commands + auth status ping itself). If > 7 days since last_seen, show advisory hint: *"It's been N days since you last authenticated to this instance — your PDI may have been released."* |
| Layer 2 | In `auth status`, make a lightweight API call (e.g. `GET /api/now/ui/meta/current`). Check for valid JSON with expected structure. If it fails or returns HTML, mark as unverified. |
| Hint behavior | Advisory only — does not block anything. Shows only when >7 days AND the API ping fails. |

### Profile Management (#87, #88)

| Decision | Resolution |
|----------|------------|
| Add `jsn profiles create` | New subcommand: `jsn profiles create <name> --instance <url> [--username ...] [--password ...]` |
| Deduplicate on login | `jsn auth login` should check if a profile already exists for that instance URL. If found, update credentials for the existing profile instead of creating a duplicate. |
| Fix `profiles list` output | Add a proper table showing: profile name, instance URL, auth status, default/active markers |
| OAuth URL leak | Keep `--print-url` but ensure sensitive info is not unnecessarily visible in shared terminal contexts |
